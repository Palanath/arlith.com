Information

What is this?

Arlith is a communication platform; an online 'place' where people can chat with each other.

How it Works

Two users download the client and create an account, then friend each other in Arlith. They can then create private communication channels with each other that they can use to communicate through. Two users can friend each other by username.

Arlith also supports a few basic profile features, such as a customizable profile icon.

Basic Features

Arlith currently supports styled direct messaging. You can embolden, italicize, change the font, or color of any parts of messages you send to others.

There is planned support for group communication, both in "Group Threads" (similar to Skype or Discord groups) and Communities, (similar to Discord's servers). If you'd like to know more details about what's in the app or what's planned, check out some screenshots below, or the development wiki for what's upcoming. In line with the app's purpose, features like animated profile pictures are entirely free.

Purpose

Arlith was made to be as seamless and usable as possible. Judgement over what features are added to the app is made based on how useful the features are suspected to be, and details are well-considered in regard to how they affect the seamless usage of the application.

If you're looking for an established, well-popularized platform, you're in the wrong place, but if you're in need of a platform that will actually let you effectively and easily communicate with others, this app is for you.

Recent Updates

Logging API - 12/23/2022

The program now implements a cleaner logging API. Logging is used for debugging, primarily; informational messages are printed somewhere whenever something important happens and when an unexpected error occurs. Developers can use log messages when they run the app or when a tester shares log messages with the developer, and can try and pinpoint what went wrong or how the program is behaving.

The new log system adds prefixes to log messages including the timestamp of when the message was logged as well as some additional information about what part of the code the log message is occurring on. The new logging system includes cleaner abstractions and a more modular way of customizing logging behavior, so the application can be more easily set up to log messages to different outputs, e.g. error messages to a file, or to disable debug messages when not needed.

File Logging - 12/25/2022

File logging is now enableable through a command line flag. Options such as the log output directory can be configured through the command line as well.

A small CLI Flag API was also added to centralize parsing and handling command line arguments.
